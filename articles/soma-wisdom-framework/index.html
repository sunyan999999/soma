    不只是RAG — 为什么AI Agent需要「智慧框架」而非「知识库」¶

    一个会反思自己思考方式的AI记忆架构

    问题：RAG的"知识拼贴"困境¶

    2025年，RAG（检索增强生成）已经是AI Agent的标配能力。Agent通过向量相似度找到相关文档片段，注入LLM上下文，生成答案。

    这个范式有一处暗伤：相关性 ≠ 思考价值。

    当你问"如何解决增长瓶颈"时，RAG会忠诚地返回最"相似"的记忆——那些包含"增长""瓶颈"关键词的段落。但这真的是你需要的吗？

    • 相似度最高的记忆可能是："上季度增长还增长了20%..."（信息性，无启发性）
    • 真正有用的思考可能是："用第一性原理重新审视用户价值"（来自"哲学"领域，关键词不匹配但结构相同）
    • 甚至更有用的是："如果全力以赴让增长停滞，最有效的3个方法是什么？"（逆向思考，RAG绝无可能返回）

    RAG找的是"看起来像"的信息，而人类找的是"能产生新思路"的视角。

    方案：SOMA的智慧框架驱动架构¶

    SOMA（体悟式智慧架构）在RAG之上叠加了一层智慧框架——它不是检索知识，而是组织思考方式。

    核心机制是7条思维规律的双向激活：

    7条思维规律¶

    规律 核心问题 当RAG失效时
    第一性原理 最基本的要素是什么？ 剥离行业术语，回到物理学层面
    系统思维 增强/平衡回路在哪里？ 找到关键节点而非罗列原因
    矛盾分析 主要矛盾是什么？ 识别对立力量而非妥协方案
    二八法则 哪20%因素造成80%结果？ 聚焦关键少数而非平等对待
    逆向思考 如何让这件事彻底失败？ 暴露盲区而非正面分析
    类比推理 其他领域如何解决类似问题？ 跨域迁移而非限于本行
    进化视角 这个问题的历史演变路径？ 时间维度上的深层理解

    双向激活机制¶

    问题 "为什么增长停滞？"
        │
        ▼
┌── 语义方向（向量相似度）──┐     ┌── 关键词方向（FTS5 RRF）──┐
│  "用户增长从 30% 降到 5%"  │     │  触发词 → 第一性原理       │
│  "市场环境变了"           │     │  触发词 → 系统思维         │
│  "竞品开始打价格战"       │     │  触发词 → 矛盾分析         │
└──────────────────────────┘     └──────────────────────────┘
        │                                    │
        └──────────┬─────────────────────────┘
                   ▼
          双向竞争融合 → 真正的相关记忆

    两条路径独立打分，但竞争融合。一条回到传统RAG的"相似"，另一条走向思维框架的"相关"。最终胜出的记忆既要语义相近，又要对深度思考有帮助。

    自我进化闭环¶

    SOMA不只是静态框架——每次会话后，它根据结果自动调整规律权重：

    • 第一性原理分析后用户说"有启发" → weight +0.02
    • 系统思维分析后用户说"没帮助" → weight -0.02
    • 某条规律长期不被触发 → 自动发现问题并降权

    触发词自动扩展：会话中与规律共现的关键词被追踪，跨5次不同会话共现后自动提升为正式触发词。

    实现：零依赖的极简栈¶

    SOMA stack:
  SQLite WAL (持久化 + FTS5全文索引)
  + ONNX Runtime (BGE-small-zh 512维embedding)
  + FAISS (可选，向量ANN搜索)
  + LiteLLM (统一LLM接口)
  + FastAPI (REST API)
  + Vue 3 + ECharts (Dashboard)
    • 零外部服务依赖：无向量数据库、无API Key、无Docker
    • 开箱即用：pip install soma-wisdom && python -m soma
    • 数据可控：全部本地存储，无隐私风险

    数据：统计可验证的稳定性¶

    基于 v0.6.1 的综合测试报告（5轮独立运行、独立数据库）：

    维度 均值 标准差 CV% 稳定性
    综合评分 65.72 ±0.04 0.07% ● 稳定
    记忆能力 96.78 ±0.15 0.15% ● 稳定
    智慧推理 76.04 ±0.05 0.07% ● 稳定

    223个单元测试全部通过，零回归。

    与ChromaDB的活体竞品对比（同数据、同查询）：

    系统 Recall@5 Recall@10 Q-P50
    SOMA 30.0% 27.5% 10.10ms
    ChromaDB 2.5% 5.0% 1.22ms

    SOMA召回率是纯向量检索的12倍——因为关键词+语义的双路径搜索更适配中文场景。

    进化：自发现新规律的元认知¶

    SOMA的 DBSCAN 聚类分析会持续扫描记忆：

    1. 找到主题高度集中的记忆簇
    2. 通过LLM分析提炼新的思维规律
    3. 人工审核后加入框架

    这不是预设的规则系统——它会自己发现人类不知道的思考方式。

    开始使用¶

    pip install soma-wisdom
python -m soma

    或从源码：

    git clone https://github.com/sunyan999999/soma.git
cd soma/soma-core
pip install -e .
python -m soma

    结语¶

    RAG解决了"如何找到信息"，SOMA在尝试解决"如何产生洞见"。

    信息检索的终点是找到，智慧框架的起点是思考。

    一个会思考、会记忆、会成长、会协作的 AI 大脑系统

    一句话说清楚 SOMA 是什么¶

    SOMA 是一个给 AI 装上的「大脑操作系统」。

    现在的 AI 大模型就像一个人拥有海量知识，但每次对话结束就「失忆」。下次再聊，你们又是陌生人。

    SOMA 解决了这个问题。 它让 AI 拥有：

    • 记忆——记住跟你聊过的每一件事，下次自动回想起来
    • 思考——用 7 种思维方式拆解问题，不是生硬匹配关键词
    • 成长——用得越久越懂你，自动调整思维偏好
    • 协作——多个 AI 分身组成团队，像人类专家团队一样讨论决策

    你可以把 SOMA 理解为一个「会思考的记忆库」，或者一个「越用越聪明的 AI 助手底座」。

    SOMA 能做什么¶

    🧠 让 AI 记住，而不是忘记¶

    普通 AI 每次对话都是全新开始。接入 SOMA 后，AI 能记住你们之前讨论过的所有事情——你的偏好、你的项目背景、你否定过的方案、你认同过的想法。下次对话时，相关记忆会自动浮现。

    💡 让 AI 思考，而不是匹配¶

    普通 AI 面对问题，做的事情是搜索匹配最相关的信息。SOMA 的做法不同：它用 7 种思维规律来拆解问题——从第一性原理、系统关联、矛盾权衡、时间演化、信息完整性等多个维度同时分析，给出更深入、更结构化的思考结果。

    🌱 让 AI 越用越聪明¶

    SOMA 会观察自己的思考效果，当一个思维角度反复帮你找到好答案，它的权重会自动上升；当某个角度一直没帮助，它的权重会下降。这不是人工调参，而是系统在真实使用中自动进化。

    🤝 让 AI 团队协作¶

    一个 AI 再强也有盲区。SOMA v0.9.0 支持创建多个 AI 专家——法律专家、技术顾问、管理教练……每个都有自己的独立记忆和专长领域。问题来了自动分给最合适的专家，多个专家意见不一致时自动协调达成共识。

    能力详解¶

    第一层：记忆底座 — 让 AI 拥有不会遗忘的大脑¶

    自动记忆¶

    每当 AI 和你对话，SOMA 会自动记录关键信息。不需要你手动保存，不需要你说「记住这个」。系统会自动判断什么值得记、什么可以忽略。

    一个例子：你告诉 AI 「我们项目用的是 React 18，数据库是 PostgreSQL，下周五要上线」。SOMA 自动拆分并记住这三条关键信息。两周后你问「我们项目用的什么数据库」，AI 直接回答「PostgreSQL」，因为记忆自动被检索出来了。

    智能去重¶

    同样的信息不会反复存储。SOMA 会判断新内容是否和已有记忆重复——如果是在更新而非新增，如果一模一样则直接跳过。这让记忆库保持精简，不会随着使用时间膨胀到无法管理。

    双重搜索¶

    当你提问时，SOMA 同时用两种方式检索记忆：

    • 关键词匹配：快速找出包含你提到词语的记忆，适合精确查找
    • 语义理解：理解你问题的含义，找出意思相关的记忆——哪怕用词完全不同

    两种结果智能融合排序，确保最相关的记忆排在前面。

    一个例子：你之前讨论过「用户登录的安全性」，现在问「怎么防止账号被盗」。用词完全不同，但语义搜索能理解两件事高度相关，把之前的讨论调出来。

    记忆整理与遗忘¶

    人的大脑不会记住所有事——不重要的事情会被遗忘，相关的事情会被合并。SOMA 模拟了同样的机制：

    • 合并摘要：当多条记忆都在说同一件事的不同方面时，系统自动将它们合并成一条更完整的记忆，消除冗余
    • 主动遗忘：长期不访问的低价值记忆会被自动归档（不是删除，需要时可以找回）
    • 智能保留：重要的、经常访问的、近期记录的记忆会被保护，不会被遗忘

    记忆安全隔离¶

    在多人使用或多 AI 共存的场景中，记忆不会串。SOMA 有三层隔离：

    • 你的记忆只有你能看到（除非你主动共享）
    • 团队共享区：你可以把某些记忆开放给同一团队的人
    • 全局知识库：通用的、不涉及隐私的知识对所有人开放

    这就像公司的网盘——有你的私人文件夹，有部门共享文件夹，也有全公司都能看的公共资料库。

    第二层：深度推理 — 不只是找答案，而是理解问题¶

    这是 SOMA 区别于普通记忆库的核心能力。它不只是「找到相关信息拼在一起」，而是用一套完整的思维框架来拆解和分析问题。

    七种思维规律¶

    SOMA 内置了 7 条思维规律，每一条代表一种分析问题的角度：

    思维规律 通俗解释 什么时候最有用
    第一性原理 回到问题最根本的地方，问「这件事本质上是什么」 面对复杂问题时，剥离表象直达本质
    系统思维 把问题放在更大系统里看，考虑各部分怎么相互影响 分析一个改动会带来哪些连锁反应
    矛盾权衡 识别问题中的对立力量，不是非此即彼，而是找平衡点 两个方案各有利弊，需要综合权衡
    时间演化 考虑事情会怎么随时间变化——短期效果 vs 长期后果 做决策时评估未来影响
    信息完整性 审视自己知道什么、不知道什么，主动补全缺失的信息 信息不足时避免武断决策
    多视角采样 站在不同角色立场看问题，避免单维度思维 涉及多方利益时需要换位思考
    行动导向 思考的最终目标是行动——结论必须能落地执行 需要产出具体方案而不是空谈分析

    一个实际例子：用户问「我应该把单体应用拆成微服务吗？」

    • 普通 AI 回答：「微服务有这些好处……也有这些坏处……」
    • SOMA 的思考过程：
    • 第一性原理：这个应用的核心价值是什么？微服务是手段不是目的
    • 系统思维：拆分会影响团队结构、部署流程、监控体系——不只是技术问题
    • 矛盾权衡：开发速度 vs 运维复杂度，团队独立性 vs 全局一致性
    • 时间演化：现在用户量 1000，半年后可能 10 万——现在拆还是以后拆？
    • 信息完整性：我还不知道你的团队规模、技术栈、预算——在哪些信息缺失的情况下给出建议？
    • 多视角：开发者怎么看、运维怎么看、产品经理怎么看、老板怎么看？
    • 行动导向：不管拆不拆，下一步具体做什么？

    最终给用户的不是一段泛泛而谈，而是一个有结构、有深度、有可操作步骤的系统分析。

    因果推理¶

    SOMA 不只记录「发生了什么」，还分析「为什么会发生」。

    如果你在排查一个 bug：「用户登录失败」，SOMA 不只是把之前关于登录的讨论调出来，而是构建因果链——上次登录模块改动→用了一个新依赖→那个依赖在 Node 18 下有兼容问题→你正好昨天升级了 Node。因果链帮助从根源解决问题，而不是反复打补丁。

    跨领域类比¶

    当当前领域找不到直接答案时，SOMA 会尝试从其他领域寻找类似案例。比如你问「怎么设计一个推荐算法」，SOMA 可能从「商店怎么摆放商品引导顾客」这个完全不同领域的讨论中找到可迁移的思路。

    质量自评¶

    每次思考结束后，SOMA 会评估自己的推理质量——逻辑是否严谨、论据是否充分、有没有遗漏关键视角。如果质量不达标，会主动标记出来建议进一步验证。

    第三层：持续进化 — 用得越久越懂你¶

    权重自动演化¶

    7 条思维规律不是平等使用的。对于你的具体场景，某些思维方式可能特别有效，某些可能不太适用。SOMA 会自动观察：

    • 哪些思维角度在过去的对话中帮到了你（你认同的、采纳的、深挖的）
    • 哪些思维角度很少用到或效果不佳

    然后自动调整各条规律的权重。比如你做产品设计工作，「系统思维」和「多视角采样」的权重可能会自动上升，而「行动导向」保持中等——因为你的场景更需要深度分析，而不是追求快速执行。

    这个过程完全自动，不需要你手动调任何参数。

    技能固化¶

    当你反复和 SOMA 讨论某类问题时，SOMA 会从成功经验中提炼出可复用的技能。比如你经常做代码审查，SOMA 会自动形成「代码安全审查」「性能瓶颈分析」「架构合理性评估」等子技能，下次遇到类似问题时直接调用。

    目前生产环境中（零熵智库平台），经过一个多月的运行，系统已自动固化了 281 个技能。

    遗忘无用信息¶

    和人类记忆一样，SOMA 不会无限堆积信息。系统会定期执行「遗忘扫描」：

    • 长期未被访问的记忆 → 归档
    • 被新信息取代的旧记忆 → 降权
    • 重要性低且时间久远的 → 谨慎移除

    这保证记忆库始终保持在可管理的大小，不会因为时间推移变得臃肿迟钝。

    第四层：群体协作 — 一个 AI 团队为你工作¶

    v0.9.0 最大的升级，是从单个 AI 的智能升级到多个 AI 协作的集体智能。

    组建专家团队¶

    你可以创建多个 AI 分身，每个有自己的专长领域和独立记忆：

    • 法律专家分身：专门处理合同、合规、知识产权相关问题
    • 技术顾问分身：专注技术架构、性能优化、代码审查
    • 产品经理分身：擅长需求分析、用户体验、功能规划
    • 管理教练分身：帮助团队建设、流程优化、战略决策

    每个分身都拥有独立的记忆和独立演化的思维权重，真正成为各自领域的专家。

    智能路由 — 问题自动找对人¶

    当你提出问题，系统会自动判断应该由哪个专家来处理：

    • 第一层判断：分析问题中的关键词——提到「合同」「违约」「赔偿」→ 自动路由给法律专家
    • 第二层判断：如果关键词不够明确，就理解问题的含义——「我们这样写会不会被人告？」虽然没有法律关键词，但语义上和法律风险高度相关
    • 第三层兜底：如果实在判断不了，交给通用分身处理

    这个过程毫秒级完成，你完全感知不到背后有多个 AI 在待命。

    共识机制 — 多种意见合而为一¶

    面对复杂问题，可能需要多个专家从不同角度共同评估。比如「公司应该自建 AI 团队还是外包？」，这涉及技术、管理、法律、财务等多个维度。

    SOMA 的做法：

    1. 同时请教技术顾问、管理教练、法律专家
    2. 每个专家从自己的角度独立给出分析
    3. 系统自动检测大家的意见有多大分歧
    4. 如果意见基本一致 → 直接合并输出
    5. 如果有中等分歧 → 用另一个 AI 阅读各方观点后进行仲裁
    6. 如果分歧严重（比如法律要求自建、技术建议外包）→ 启动辩证综合，找到更高层次的统一方案
    7. 最终输出一个综合了多方意见的结论，同时标注哪些点存在争议、少数派的观点是什么

    你看到的最终结果，是一个经过多专家交叉验证的、更全面更可靠的建议。

    分布式演化 — 每个专家各自成长¶

    每个专家分身在自己的领域里独立演化思维权重：

    • 法律专家在法律对话中，发现「矛盾权衡」特别重要（法律问题往往要权衡各方利益），权重自动上升
    • 技术顾问在技术对话中，发现「第一性原理」最有用（技术问题需要回归根本），权重自动上升

    然后定期把各专家的「经验」合并成全局智慧，再回传给所有专家。这就像每个部门的经验汇总成公司知识库——个人成长和集体成长相互促进。

    应用场景¶

    场景一：企业智能客服¶

    痛点：客服系统只能回答固定 FAQ，遇到复杂问题转人工。300 个客服每天回答大量重复问题。

    SOMA 方案： - 记忆底座记住所有历史工单和解决方案 - 7 种思维规律帮 AI 理解用户真实的意图（不只是匹配关键词） - 多个领域分身——技术咨询、投诉处理、售后支持——自动路由 - 随着使用不断进化，回答越来越精准 - 客服人员减少 40%，用户满意度提升 25%（基于零熵智库生产数据推算）

    场景二：个人 AI 知识管家¶

    痛点：每天接收大量信息——会议记录、文章阅读、项目讨论——但无法有效沉淀。过几个月就忘了当时是什么决策、什么原因。

    SOMA 方案： - 自动记录你的重要对话和决策 - 当你在新对话中提到相关话题时，旧记忆自动浮现 - 随着使用时间增长，系统越来越懂你的工作方式和偏好 - 你的知识资产不会丢失，而是持续增值

    一个真实使用场景：你在 3 个月前和技术团队讨论过一个架构方案，当时否决了方案 A 选择了方案 B。3 个月后新同事问「为什么不用方案 A」，你直接问 SOMA，它把当时的讨论、否决原因、权衡过程全部调出来。

    场景三：团队协作 AI 中台¶

    痛点：团队知识分散在不同人的聊天记录、文档、邮件里。新人入职需要 3 周才能熟悉项目。关键决策的上下文经常丢失。

    SOMA 方案： - 为团队创建共享记忆区——所有的人讨论、决策、经验都沉淀下来 - 为每个成员创建私有 AI 分身——你的偏好、你关注的方向被单独记住 - 新成员入职时，AI 助手直接回答「这个项目为什么选了 React 而不是 Vue」「上次架构评审的结论是什么」 - 团队成员离开时，他的知识留在系统中而不是带走 - 从「人找知识」变成「知识找人」

    场景四：教育辅导平台¶

    痛点：一个老师面对 50 个学生，无法个性化教学。每个学生的薄弱点和学习风格不同。

    SOMA 方案： - 为每个学生创建学习分身——记住他/她掌握什么、卡在哪里、喜欢什么学习方式 - 数学、语文、英语各有专家分身，根据学生状态自动调整教学策略 - 教学分身随着学生进步不断演化——发现某个讲解方式对这个学生特别有效，权重自动上升 - 老师不再疲于应对 50 种需求，AI 助手承担个性化辅导，老师聚焦教学设计

    场景五：法律/医疗/金融专业顾问¶

    痛点：专业领域知识更新快、案例量大、从业者需要快速检索和分析。

    SOMA 方案： - 记忆层承载海量案例、法条、文献——过目不忘 - 推理层用 7 种思维规律分析案件——不只是找相似案例，而是系统推导 - 因果链理清「什么行为导致什么法律后果」 - 多专家协作：同一案件，刑事角度和民事角度分别分析后综合结论 - 每次分析都是一次学习，系统越来越擅长你的专业领域

    对不同用户的价值¶

    对个人用户¶

    场景 没有 SOMA 有了 SOMA
    使用 AI 助手 每次对话都是新人，需要反复解释背景 AI 记住你的所有偏好和历史，越用越贴切
    知识管理 信息散落在各处，想找找不到 自动沉淀、智能检索，需要时自动浮现
    复杂决策 得到泛泛的建议，缺乏深度分析 7 维思维框架系统分析，给出有结构的深度建议
    学习成长 学过就忘，无法积累 每次讨论都沉淀为可检索的经验

    一句话总结：SOMA 让你的 AI 助手从「金鱼记忆的实习生」变成「越用越懂你的资深搭档」。

    对团队和中小企业¶

    场景 没有 SOMA 有了 SOMA
    知识传承 依赖口头传授和散落的文档 团队知识自动沉淀，新人即问即答
    决策质量 拍脑袋决定，缺乏系统分析 多角度推理 + 历史经验回溯
    团队协作 信息孤岛，重复讨论同样问题 共享记忆 + 多专家协作，信息透明
    人员流动 人走了知识就没了 知识留在系统里，不依赖具体的人
    成本投入 需要多个 SaaS 工具拼凑 一套系统覆盖记忆+推理+协作

    一句话总结：SOMA 是团队的知识地基——所有人的经验持续沉淀、所有人的决策有章可循。

    对企业¶

    价值维度 具体体现
    效率提升 客服、法务、技术支持等知识密集型岗位的效率大幅提升
    知识资产化 企业最重要的知识不再存在员工的脑子里，而是存在系统中
    决策科学化 从「凭经验拍板」到「多维分析+历史验证」
    降低 AI 门槛 不需要 AI 专家团队，一个 pip install 就拥有完整认知架构
    数据安全 全部本地部署，数据不出企业边界，适合涉密场景
    成本可控 零外部依赖，不需要 Redis/消息队列/注册中心，运维成本极低

    特别适合： - 知识密集型行业（法律、金融、医疗、咨询） - 有大量重复决策场景的企业（客服、审核、合规） - 注重数据安全和隐私保护的机构（政府、军工、医院） - 希望用 AI 提升效率但缺乏 AI 技术团队的中小企业

    与其它方案的区别¶

    SOMA vs 普通向量数据库（ChromaDB / Pinecone / Milvus）¶

    向量数据库 SOMA
    能做什么 存储和搜索文本 存储 + 搜索 + 推理 + 演化 + 协作
    记忆方式 依赖相似度匹配 关键词 + 语义双通道，自动去重和合并
    思考能力 无 7 条思维规律深度推理
    自我进化 无 自动调整权重、固化技能、遗忘无用信息
    多用户隔离 需要自己实现 内置三态隔离（私有/组/全局）
    部署 Docker 或云服务 pip install 一行命令

    简单说：向量数据库是「硬盘」，SOMA 是「大脑」——硬盘能存数据，但不会思考。

    SOMA vs 其他 Agent 框架（LangChain / CrewAI / AutoGen）¶

    其他 Agent 框架 SOMA
    定位 帮开发者快速搭建 LLM 应用 给 AI 装上认知大脑
    推理方式 依赖 prompt 工程，需要人工设计 内置 7 维思维框架，自动拆解
    记忆 基础的对话历史或简单向量检索 完整的记忆生命周期：存储→合并→遗忘→检索
    进化 无，权重需要人工调整 自动演化，越用越好
    多 Agent 预设脚本流程 动态路由 + 共识协议 + 独立演化
    上手难度 需要理解 agent 概念和框架设计 remember() + respond() 两个核心方法

    简单说：其他框架是「给你零件自己组装」，SOMA 是「给你一个已经会思考的大脑」。

    开始使用¶

    最快体验（30 秒）¶

    pip install soma-wisdom

    第一次使用（3 行代码）¶

    from soma import SOMA

# 创建一个 AI 大脑
soma = SOMA(use_vector_search=True)

# 让它记住一件事
soma.remember("我们项目后端用 Go 语言，前端用 React，数据库用 PostgreSQL")
# 返回: memory_id

# 向它提问
answer = soma.respond("我们应该用什么技术栈？")
print(answer)
# → "根据之前的信息，你们项目使用 Go 作为后端语言、React 作为前端框架、
#    PostgreSQL 作为数据库。这个技术栈的组合在性能、开发效率和生态方面
#    都比较均衡……"

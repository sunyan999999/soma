# 常见问题

## 基础

### SOMA 是什么？

SOMA 是一个认知框架，而不是向量数据库。它在检索记忆之前先通过 7 条思维规律分解问题——让 AI Agent 系统地分析，而非简单的模式匹配。

### 和 Mem0 / ChromaDB 的区别？

| | Vector DB | Mem0 | SOMA |
|---|---|---|---|
| 存储和检索 | ✓ | ✓ | ✓ |
| 推理框架 | ✗ | ✗ | ✓ 7 条思维规律 |
| 权重自演化 | ✗ | ✗ | ✓ |
| 因果推理 | ✗ | ✗ | ✓ |
| 冲突检测 | ✗ | ✗ | ✓ |
| 离线运行 | 部分 | ✗ | ✓ ONNX + SQLite |

详见 [SOMA vs Mem0](articles/soma-vs-mem0.md) 和 [SOMA vs Letta](articles/soma-vs-letta.md)。

### 需要 OpenAI API Key 吗？

不需要。SOMA 默认使用 ONNX 本地模型进行嵌入，零外部依赖。你也可以配置任意 LLM（OpenAI / Anthropic / Ollama / 本地模型）。

## 安装与配置

### 如何安装？

```bash
pip install soma-wisdom
```

Python 3.10+ 即可。首次运行自动下载 ONNX 模型（~100MB）。

### 支持哪些 LLM？

通过 `llm_config.json` 配置，支持所有 OpenAI 兼容接口：

```json
{
  "provider": "openai",
  "model": "gpt-4o",
  "api_key": "sk-...",
  "base_url": "https://api.openai.com/v1"
}
```

也可使用 Anthropic、Ollama 或完全离线模式。

### 数据存储在哪里？

默认 `~/.soma/soma-core/`，SQLite 数据库。数据库路径可通过 `SOMAConfig` 自定义。

## 使用

### SOMA 如何处理记忆遗忘？

采用艾宾浩斯遗忘曲线：记忆根据时间衰减，低重要性记忆会被主动归档。v0.7.0 起支持相似记忆自动摘要合并。

### 7 条思维规律是什么？

1. **第一性原理** — 分解到基本元素
2. **系统思维** — 要素间关系与反馈
3. **批判性思维** — 质疑假设与证据
4. **类比推理** — 跨域结构映射
5. **逆向思维** — 从目标倒推
6. **抽象思维** — 通用模式提取
7. **实用主义** — 可操作性与反馈

规律权重根据使用效果自动演化。

### 支持中文吗？

完全支持。双语关键词提取、中英否定词检测、中文文档。零熵智库的生产环境全部使用中文记忆。

### 记忆数量上限？

单机可处理 10 万+ 条记忆，检索延迟保持在 10ms 以内（v1.0 目标）。当前生产环境已验证 1752 条记忆稳定运行。

## 安全与隐私

### 数据会发送到云端吗？

不会。SOMA 本地优先。嵌入使用本地 ONNX 模型，记忆存储在本地 SQLite。只有在你主动配置外部 LLM 时才会发送 API 请求。

### 如何保护敏感记忆？

通过 `user_id` 实现数据隔离。不同用户的记忆完全独立。企业版（v1.0 计划）将支持 RBAC 和审计日志。

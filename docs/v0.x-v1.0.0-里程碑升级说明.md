# SOMA v0.x → v1.0.0 里程碑升级说明

> 写给所有已部署 SOMA 的用户（零熵智库 Qoder 等）。
> v1.0.0 = v0.10.0 重命名为里程碑版本。五条能力线（记忆/推理/协作/进化/工程）完整交汇。

## 一、v1.0.0 的核心意义

**v1.0.0 不是功能堆砌出来的版本号。** 它是 SOMA 从"实验性工具"到"可生产认知内核"的标志。

| 衡量维度 | v0.9.1（上次发布） | v1.0.0（里程碑） |
|----------|-------------------|-------------------|
| 测试数量 | 485 | **511** |
| 综合基准 | 80.5 | **85.5** |
| 记忆层数 | L1（情节） | **L1+L2+L3（情节+场景+画像）** |
| 多Agent | 基础协作 | **编排器+路由+共识** |
| API方法 | ~16 | **~30** |
| MCP工具 | ~8 | **~13** |

## 二、从 v0.9.1 升级到 v1.0.0

### 2.1 零改动升级（默认行为不变）

```bash
pip install --upgrade soma-wisdom==1.0.0
```

```python
# v0.9.1 代码无需任何修改
from soma import SOMA
soma = SOMA(llm="deepseek-chat")
soma.remember("内容")
soma.respond("问题")
soma.chat("问题")
```

**所有新功能默认关闭。** 不主动开启 = 行为与 v0.9.1 完全一致。

### 2.2 SOMA() 新增可选参数

```python
# v1.0.0 完整签名（所有新增参数在末尾，带默认值）
SOMA(
    framework_config=None,    # 不变
    llm="deepseek-chat",      # 不变
    llm_api_key="",           # 新增: LLM API Key
    llm_base_url="",          # 新增: 自定义 base_url
    use_vector_search=True,   # 不变
    persist_dir=None,         # 不变
    recall_threshold=0.01,    # 不变
    top_k=5,                  # 不变
    agent_id="",              # 不变
    group_id="",              # 不变
    orchestration_mode="single",     # 新增: "single"|"multi"
    scene_extraction_enabled=False,  # 新增: L2场景提取
    profile_extraction_enabled=False,# 新增: L3画像提取
)
```

### 2.3 remember() 新增可选参数

```python
# 旧调用方式完全兼容
soma.remember("内容")

# 新增 auto_capture 参数（最末尾，默认 None）
soma.remember("内容", auto_capture=True)   # 单次强制触发场景提取
soma.remember("内容", auto_capture=False)  # 单次跳过场景提取
soma.remember("内容")                      # None = 尊重全局配置
```

### 2.4 stats 字典扩展（向后兼容）

```python
# v0.9.1: 4个键
stats = {"episodic": ..., "semantic": ..., "skill": ..., "indexed_vectors": ...}

# v1.0.0: 分层开启后新增3个键（均在末尾，不影响旧键）
stats = {..., "scenes": ..., "profile_entries": ..., "capture_state": ...}
```

## 三、v1.0.0 新增功能速查

### 3.1 三层记忆（L1/L2/L3）

```python
soma = SOMA()

# 启用分层记忆（默认关闭）
soma.enable_layered_memory(scene_warmup=5, profile_interval=10)

# 正常使用，场景自动聚合
soma.remember("修复了OrderService的N+1查询bug", user_id="dev1")
soma.remember("添加了数据库连接池监控", user_id="dev1")

# 查看自动生成的场景块
scenes = soma.get_scenes(user_id="dev1")
for s in scenes:
    print(s["title"], s["memory_count"])

# 查看自动生成的用户画像
profile = soma.get_profile(user_id="dev1")
for trait in profile:
    print(trait["category"], trait["value"])

# 手动触发捕获
soma.capture_scenes(user_id="dev1", force=True)
soma.update_profile(user_id="dev1", force=True)

# 白盒 Markdown 输出
print(soma.get_scene_markdown(scenes[0]["scene_id"]))
print(soma.get_profile_markdown(user_id="dev1"))

# 禁用
soma.disable_layered_memory()
```

### 3.2 多Agent编排

```python
soma = SOMA(orchestration_mode="multi")

# 注册专家
soma.register_expert("tech_expert", expertise=["技术", "架构", "性能"])
soma.register_expert("biz_expert", expertise=["商业", "市场", "增长"])

# 列出专家
for expert in soma.list_experts():
    print(expert["agent_id"], expert["expertise"])

# 多Agent求解（自动路由+共识）
result = soma.solve_multi("如何平衡技术投入和业务增长？")
print(result["answer"])
print(result["routing"])     # 路由到哪些专家
print(result["consensus"])   # 共识结果
```

### 3.3 MCP 工具（5个新增）

| 工具 | 说明 |
|------|------|
| `soma_scenes` | 搜索场景块 |
| `soma_profile` | 查看用户画像 |
| `soma_capture` | 手动触发场景提取 |
| `soma_layered_stats` | 分层统计 |
| `soma_scene_markdown` | 场景白盒文档 |

## 六、已知限制

1. **场景提取需要 LLM**：无 LLM 配置时 `capture_scenes()` 优雅降级，返回 0
2. **Scene/Profile 检索权重固定**：`scene_retrieval_weight=0.3`, `profile_retrieval_weight=0.5`
3. **Dispatch 串行执行**：多专家逐个调用，未并行化（计划 v1.1 优化）

## What SOMA Is, in Plain Language

SOMA is an "operating system for AI brains."

Today's large language models are like a person with vast knowledge — but every conversation ends with amnesia. The next time you talk, you're strangers again.

**SOMA fixes this.** It gives AI:

- **Memory** — remembers everything you've discussed, automatically surfacing relevant memories in future conversations
- **Thinking** — decomposes problems using 7 thinking patterns, not just keyword matching
- **Growth** — gets better the more you use it, automatically adjusting its thinking style to match your needs
- **Collaboration** — multiple AI personas form teams, deliberating and deciding like a panel of human experts

Think of SOMA as a "memory bank that knows how to think," or a "foundation for AI assistants that get smarter with every conversation."

## What SOMA Can Do

### 🧠 Makes AI Remember, Not Forget

A typical AI starts fresh every conversation. With SOMA, the AI remembers everything you've discussed — your preferences, your project context, solutions you rejected, ideas you approved. In the next conversation, relevant memories surface automatically.

### 💡 Makes AI Think, Not Just Match

When facing a question, a typical AI searches for and matches the most relevant information. SOMA takes a different approach: it uses **7 thinking laws** to decompose problems — analyzing from first principles, system interconnections, trade-offs, temporal evolution, information completeness, and more — all at once. The result is deeper, better-structured thinking.

### 🌱 Makes AI Smarter Over Time

SOMA observes its own thinking effectiveness. When a particular thinking angle consistently leads to good answers, its weight automatically increases. When an angle proves unhelpful, its weight drops. This isn't manual tuning — the system evolves automatically through real-world use.

### 🤝 Makes AI Teams Work Together

A single AI, no matter how capable, has blind spots. SOMA v0.9.0 lets you create multiple AI experts — a legal expert, a technical advisor, a management coach — each with independent memory and specialized knowledge. Questions are automatically routed to the most suitable expert. When experts disagree, the system coordinates to reach consensus.

## Detailed Capabilities

### Layer 1: Memory Foundation — Giving AI an Unforgettable Brain

#### Automatic Memory

Whenever AI converses with you, SOMA automatically records key information. No manual saving, no "remember this" commands. The system autonomously judges what's worth keeping and what can be ignored.

**An example**: You tell the AI "Our project uses React 18, the database is PostgreSQL, and we're going live next Friday." SOMA automatically splits and remembers these three pieces of information. Two weeks later, when you ask "What database does our project use?", the AI answers "PostgreSQL" — because the memory was automatically retrieved.

#### Smart Deduplication

The same information won't be stored twice. SOMA checks whether new content duplicates existing memories — if it's an update, it refines; if it's identical, it skips. This keeps the memory bank lean and prevents it from bloating over time.

#### Dual-Search Retrieval

When you ask a question, SOMA searches memory through two channels simultaneously:

- **Keyword matching**: quickly finds memories containing the words you mentioned — perfect for exact lookups
- **Semantic understanding**: grasps the meaning of your question and finds conceptually related memories — even if the wording is completely different

The two result sets are intelligently fused and ranked, ensuring the most relevant memories appear at the top.

**An example**: You previously discussed "user login security." Now you ask "how to prevent account theft." Completely different words, but semantic search understands the two topics are highly related and surfaces the earlier discussion.

#### Memory Organization and Forgetting

Human brains don't remember everything — unimportant things are forgotten, related things are merged. SOMA simulates the same mechanisms:

- **Consolidation**: When multiple memories cover different aspects of the same thing, the system automatically merges them into one richer memory, eliminating redundancy
- **Active forgetting**: Long-unaccessed, low-value memories are automatically archived (not deleted — they can be recovered if needed)
- **Smart retention**: Important, frequently accessed, and recent memories are protected from forgetting

#### Secure Memory Isolation

In multi-user or multi-AI scenarios, memories don't leak across boundaries. SOMA has three isolation levels:

- **Your memories are visible only to you** (unless you explicitly share them)
- **Team shared space**: you can open certain memories to teammates
- **Global knowledge base**: universal, non-private knowledge accessible to everyone

This is like a company drive — you have your private folder, department shared folders, and a company-wide public library.

### Layer 2: Deep Reasoning — Not Just Finding Answers, But Understanding Problems

This is the core capability that sets SOMA apart from ordinary memory banks. It doesn't just "find relevant information and stitch it together" — it uses a complete thinking framework to decompose and analyze problems.

#### The Seven Thinking Laws

SOMA has 7 built-in thinking laws, each representing one angle of analysis:

| Thinking Law | Plain Language | Most Useful When |
|-------------|---------------|------------------|
| **First Principles** | Go back to the root — ask "what is this, fundamentally?" | Facing complex problems, strip away surface noise |
| **Systems Thinking** | See the problem in its larger system — how do parts affect each other? | Analyzing ripple effects of a change |
| **Trade-off Analysis** | Identify opposing forces — not either/or, find the balance point | Two options each have pros and cons |
| **Temporal Evolution** | Consider how things change over time — short-term effects vs. long-term consequences | Making decisions with future impact |
| **Information Completeness** | Examine what you know and what you don't — actively fill gaps | Avoiding hasty decisions with incomplete info |
| **Multi-Perspective** | Stand in different roles' shoes — avoid single-dimension thinking | Decisions affecting multiple stakeholders |
| **Action Orientation** | The goal of thinking is action — conclusions must be executable | Need concrete plans, not just abstract analysis |

**A real example**: A user asks "Should I break my monolith into microservices?"

- Typical AI response: "Microservices have these benefits... and these drawbacks..."
- SOMA's thinking process:
  - **First Principles**: What is this application's core value? Microservices are a means, not an end
  - **Systems Thinking**: Splitting affects team structure, deployment processes, monitoring — it's not just a technical issue
  - **Trade-off Analysis**: Development speed vs. operational complexity, team autonomy vs. global consistency
  - **Temporal Evolution**: You have 1,000 users now, potentially 100,000 in six months — split now or later?
  - **Information Completeness**: I still don't know your team size, tech stack, budget — what advice can I give with missing information?
  - **Multi-Perspective**: How do developers see this? Operations? Product manager? The CEO?
  - **Action Orientation**: Regardless of splitting or not, what's the concrete next step?

What the user gets isn't generic platitudes — it's a structured, deep, actionable systematic analysis.

#### Causal Reasoning

SOMA doesn't just record "what happened" — it analyzes "why it happened."

If you're debugging: "User login is failing," SOMA doesn't just pull up past login discussions. It builds a causal chain — the login module was last modified → it pulled in a new dependency → that dependency has compatibility issues under Node 18 → you just upgraded Node yesterday. Causal chains help solve root causes instead of repeatedly applying band-aids.

#### Cross-Domain Analogy

When no direct answer exists in the current domain, SOMA tries to find analogous cases from other domains. If you ask "how to design a recommendation algorithm," SOMA might draw insights from a discussion about "how stores arrange products to guide shoppers" — a completely different domain with transferable patterns.

#### Quality Self-Assessment

After each thinking session, SOMA evaluates its own reasoning quality — is the logic sound, is the evidence sufficient, are there missed perspectives? If quality falls short, it proactively flags the result for further verification.

### Layer 3: Continuous Evolution — Gets Better the More You Use It

#### Automatic Weight Evolution

The 7 thinking laws aren't used equally. For your specific context, some thinking angles may be particularly effective, while others may be less applicable. SOMA automatically observes:

- Which thinking angles helped in past conversations (ones you agreed with, adopted, explored deeper)
- Which thinking angles were rarely useful or performed poorly

Then it automatically adjusts each law's weight. For example, if you do product design work, "Systems Thinking" and "Multi-Perspective" might automatically rise in weight, while "Action Orientation" stays moderate — because your context needs deeper analysis rather than rushing to execute.

**This process is fully automatic. You never need to tune any parameters.**

#### Skill Solidification

When you repeatedly discuss certain types of problems with SOMA, it distills reusable skills from successful experiences. For instance, if you frequently do code reviews, SOMA automatically forms sub-skills like "security code review," "performance bottleneck analysis," and "architecture soundness assessment" — directly invoked the next time similar problems arise.

In the current production environment (零熵智库 platform), after over a month of operation, the system has automatically solidified 281 skills.

#### Forgetting Useless Information

Like human memory, SOMA doesn't accumulate information indefinitely. The system periodically runs "forgetting scans":

- Long-unaccessed memories → archived
- Old memories superseded by new information → demoted
- Low-importance, timeworn memories → carefully removed

This ensures the memory bank stays at a manageable size, never becoming bloated or sluggish over time.

### Layer 4: Collective Intelligence — An AI Team Working for You

The biggest upgrade in v0.9.0: from single-AI intelligence to multi-AI collaborative intelligence.

#### Building Expert Teams

You can create multiple AI personas, each with its own expertise domain and independent memory:

- Legal expert persona: specialized in contracts, compliance, IP
- Technical advisor persona: focused on architecture, performance optimization, code review
- Product manager persona: skilled at requirements analysis, user experience, feature planning
- Management coach persona: helps with team building, process optimization, strategic decisions

Each persona has independent memory and independently evolved thinking weights — truly becoming an expert in its domain.

#### Smart Routing — Questions Automatically Find the Right Person

When you ask a question, the system automatically determines which expert should handle it:

- **Tier 1 judgment**: analyzes keywords in the question — "contract," "breach," "liability" → auto-routes to legal expert
- **Tier 2 judgment**: if keywords aren't clear enough, understands the question's meaning — "Could we get sued for doing it this way?" contains no legal keywords but is semantically highly related to legal risk
- **Tier 3 fallback**: if truly uncertain, routes to the general-purpose persona

This process completes in milliseconds. You never perceive multiple AIs standing by behind the scenes.

#### Consensus Mechanism — Merging Multiple Opinions Into One

Complex problems may need multiple experts evaluating from different angles. For example, "Should we build an in-house AI team or outsource?" touches technology, management, legal, and financial dimensions.

SOMA's approach:

1. Simultaneously consult the technical advisor, management coach, and legal expert
2. Each expert independently analyzes from their perspective
3. The system automatically detects how much opinions diverge
4. If opinions largely agree → directly merge and output
5. If moderate disagreement → use another AI to read all viewpoints and arbitrate
6. If severe divergence (e.g., legal requires in-house, technical recommends outsourcing) → launch dialectical synthesis to find a higher-level unified solution
7. Output a conclusion synthesizing multiple perspectives, with annotations on disputed points and minority views

The final result you see is a more comprehensive, more reliable recommendation — cross-validated by multiple experts.

#### Distributed Evolution — Each Expert Grows Independently

Each expert persona independently evolves its thinking weights within its domain:

- The legal expert, through legal conversations, discovers that "Trade-off Analysis" is particularly important (legal problems often require balancing multiple interests) — its weight automatically rises
- The technical advisor, through technical conversations, finds "First Principles" most useful (technical problems demand going back to fundamentals) — its weight automatically rises

Periodically, each expert's "experience" is merged into global wisdom and redistributed back to all experts. It's like each department's experience being aggregated into the company knowledge base — individual growth and collective growth reinforce each other.

- Teaching personas continuously evolve with student progress — when a particular explanation style proves especially effective for a student, its weight automatically rises

### Scenario 5: Legal / Medical / Financial Professional Advisor

**Pain point**: Professional domain knowledge updates rapidly, case volume is massive, practitioners need fast retrieval and analysis.

**SOMA's approach**:
- Memory layer carries vast volumes of cases, statutes, and literature — photographic memory

## How SOMA Differs From Other Solutions

### SOMA vs. Vector Databases (ChromaDB / Pinecone / Milvus)

| | Vector Database | SOMA |
|---|----------------|------|
| What it does | Store and search text | Store + search + reason + evolve + collaborate |
| Memory approach | Relies on similarity matching | Keyword + semantic dual-channel, auto-dedup and merge |
| Thinking capability | None | 7 thinking laws for deep reasoning |
| Self-evolution | None | Auto-adjust weights, solidify skills, forget useless info |
| Multi-user isolation | Must implement yourself | Built-in three-state isolation (private/group/global) |
| Deployment | Docker or cloud service | `pip install` — one command |

**In short**: Vector databases are "hard drives." SOMA is a "brain." A hard drive stores data, but it doesn't think.

### SOMA vs. Agent Frameworks (LangChain / CrewAI / AutoGen)

| | Other Agent Frameworks | SOMA |
|---|----------------------|------|
| Positioning | Help developers quickly build LLM applications | Give AI a cognitive brain |
| Reasoning approach | Depends on prompt engineering, needs manual design | Built-in 7-dimension thinking framework, automatic decomposition |
| Memory | Basic conversation history or simple vector retrieval | Complete memory lifecycle: store → merge → forget → retrieve |
| Evolution | None, weights need manual adjustment | Automatic evolution, gets better with use |
| Multi-Agent | Pre-scripted workflow | Dynamic routing + consensus protocol + independent evolution |
| Learning curve | Need to understand agent concepts and framework design | Two core methods: `remember()` + `respond()` |

**In short**: Other frameworks give you "parts — assemble it yourself." SOMA gives you "a brain that already knows how to think."

## Getting Started

### Quickest Experience (30 seconds)

```bash
pip install soma-wisdom
```

### First Use (3 lines of code)

```python
from soma import SOMA

# Create an AI brain
soma = SOMA(use_vector_search=True)

# Make it remember something
soma.remember("Our project uses Go for backend, React for frontend, PostgreSQL for database")
# Returns: memory_id

# Ask it a question
answer = soma.respond("What tech stack should we use?")
print(answer)
# → "Based on previous information, your project uses Go as the backend language,
#    React as the frontend framework, and PostgreSQL as the database. This tech
#    stack combination is well-balanced in terms of performance, development
#    efficiency, and ecosystem..."
```

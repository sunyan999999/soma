# 案例研究

## 零熵智库 — 生产环境中的 SOMA

**场景**: 企业级 AI 知识管理系统，1752 条真实业务记忆

**部署方式**: SOMA + FastAPI Dash 面板，本地 ONNX 嵌入（零外部 API 依赖）

**关键数据** (v0.8.0 基准测试):

| 指标 | 数值 |
|------|------|
| 语义召回率 | 100% |
| 综合评分 | 80.5/100 |
| 查询延迟 | 209ms |
| 基准测试耗时 | 66s |
| 系统负载 | 0.13 |
| 测试覆盖 | 422/422 全通过 |

**核心收益**:

1. **从检索到推理**: v0.8.0 的因果推理引擎让系统能回答"为什么"类问题，而非仅做关键词匹配
2. **冲突检测**: 自动标记矛盾的商业分析结论（如"降价→用户增长"vs"降价→品牌贬值"）
3. **跨域类比**: 将经济学原理自动关联到产品策略，发现非直觉的洞察

**运维经验**:

- 冷却保护机制：基准测试端点三层防护（运行检查409 + 冷却间隔300s + 超时600s），防止重复触发耗尽资源
- CPU 优化：批量 ONNX 编码（单次替代逐条）将系统负载从 0.97 降至 0.13
- 版本自动检测：`importlib.metadata.version()` 替代硬编码，dash 面板始终显示正确版本

## Glaude — AI Coding Agent 集成 SOMA

**场景**: 用 Go 实现的 Claude Code 替代品，集成 SOMA 作为开发者记忆系统

**集成方式**:

```python
# 会话启动时检索相关记忆
python c:/SOMA/soma-core/examples/dev_memory.py recall "<任务关键词>"

# 关键事件记录
python c:/SOMA/soma-core/examples/dev_memory.py save -i 0.9 "决定: X"
python c:/SOMA/soma-core/examples/dev_memory.py save -i 0.95 "约束: X"
```

**记忆类型**:

| 类型 | 重要性 | 示例 |
|------|--------|------|
| 设计决策 | 0.9 | "选择批处理而非流式处理，因为..." |
| Bug 记录 | 0.9 | "零值返回的根因是..." |
| 项目约束 | 0.95 | "禁止 CGO，必须保持交叉编译" |
| 用户纠正 | 0.85 | "不要用 emoji，保持简洁" |

**收益**:

- 跨会话保持项目上下文，无需每次重新解释架构
- 错误模式积累：同类错误自动关联历史修复方案
- CLAUDE.md 与 SOMA 记忆双重保障：静态指令 + 动态经验
